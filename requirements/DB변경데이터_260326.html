점검이력 UI/UX 변경

- 점검이력은 표형태로 제공한다.
- 등록, 대기중, 완료된 모든 스케줄이 보이고, 표는 체크박스, 스케줄ID, 스케줄명, 점검룰, 점검상태, 소요시간, 최근종료일시, 다음시작일시, 점검자 헤더를 가진다
- 표 위에는 Text 검색이 있으며 Text검색은 스케줄명, 점검룰, 등록자가 검색되고 옆에 상세필터가 있고, 상세필터 버튼을 누르면 점검상태 검색 제공
- 점검상태는 대기, 점검중, 완료, 실패, 중지 가 있다.
- 스케줄을 체크 하고 오른쪽 상단에 `중지` 버튼을 누르면 선택된 스케줄이 중지 된다(중지 할것인지 확인한다)
- 스케줄을 체크 하고 오른쪽 상단에 `재시작` 버튼을 누르면 선택된 스케줄이 재시작 된다(재시작 할것인지 확인한다)
- 스케줄을 체크 하고 오른쪽 상단에 `삭제` 버튼을 누르면 선택된 스케줄이 삭제 된다(삭제 할것인지 확인한다)
- 삭제는 스케줄 취소를 의미하며, 등록된 예약 점검 작업이 있다면 전부 삭제하며, 취소 상태는 관리하지 않는다
- 스케줄은 재사용하며, 점검상태, 종료일시, 다음시작일시는 업데이트 된다.
- 스케줄을 선택하면 스케줄의 기본정보와, 점검대상 목록이 보이고, 수정버튼을 클릭하면 스케줄 수정 페이지로 이동한다.
- 표의 상단 오른쪽에 `스케줄 등록` 버튼을 제공하여 스케줄을 신규로 등록 하는 페이지로 이동한다.
- 표의 상단에는 스케줄ID 기준 `대기, 점검중, 점검완료, 중지,실패` 를 나타내며, 선택시 필터링이 된다

조치계획 UI/UX

- 최소 검출단위(컬럼+룰) 의 상태를 기준으로 신청 및 관리를 진행
- 사용자가 조치대상이 되는 검출단위를 체크하고 검출목록 위의 조치계획작성 버튼 클릭
- 조치계획작성 버튼을 클릭하면 조치대상목록에 선택된 검출단위목록, 검출원인, 조치방법, 조치완료일, 주의사항 이 보인다
- 검출원인을 작성한다
- 조치방법 을 작성한다
- 조치완료일 을 지정한다
- 주의사항은 : 조치계획서를 작성하고, 출력하여 담당팀장 결재 후 업로드 하여야 합니다. 문구를 표시
- `저장` 버튼을 클릭한다.
- `저장` 버튼 클릭시 조치계획 관리 페이지로 이동한다.
- 조치계획 관리 페이지에는 조치계획서 목록이 있다.
- 조치계획서 목록 테이블
-- 조치계획서ID|상태|작성일시|작성자|DB명|조치대상수|조치완료일|조치방법|이행점검결과 개수
-- 상태 설명
--- 등록 대기 : 조치계획서 등록 대기 상태
--- 등록 완료 : 조치계획서 업로드 완료
--- 조치 완료 : 이행점검 후 검출결과가 0 인 경우
-- 조치계획서 를 선택하면 모달 팝업으로 조치대상(검출경로|건수|이행점검일시|이행점검 결과 수), 조치방법, 조치완료일, 조치계획서 업로드 가 보이고, 하단에 조치계획서 다운로드, 하단 오른쪽에 이행점검 버튼이 있다
- 조치계획서 다운로드 하면 word 파일로 다운로드 되며 word안에 제공가능한 정보가 채워진채로 다운로드 된다.
-- 조치계획서 word 내용

--- 업무명
--- 작성자 
--- 조치대상
--- 검출원인
--- 조치방법
--- 조치완료일
-- 제공가능한 정보
--- 부서명

--- 조치대상
--- 검출원인

- 업로드는 여러파일 업로드 가능하며 pdf 파일만 업로드로 제한한다.
- 이행점검 버튼을 클릭하면 조치대상을 전부 이행점검하며,  이행점검일시와 이행점검 결과 수가 업데이트된다
- 전체 검출경로가 이행점검 결과 수가 모두 0이 되면 상태가 조치완료 로 변경된다
- 일반사용자는 본인이작성한 조치계획서만 보인다.
- 관리자는 모든사용자의 조치계획서가 보인다
- 관리자 화면의 조치계획서 오른쪽 상단에 `쪽지발송` 을 클릭하면 조치예정일이 지났는데 이행점검 결과 수가 0> 인애들에게 쪽지가 발송된다.
- 표의 상단에는 `조치대상, 조치완료, 등록대기, 등록완료` 를 나타내며 클릭시 필터링이 된다
