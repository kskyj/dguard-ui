# Mock vs 외부 구현 검증 체크리스트

각 항목을 위에서 아래로 수행. 외부 페이지에서 동작/구조를 확인하고 mock과 비교.

## 1. 레이아웃 / 화면 구조
- [ ] 좌측 사이드바 폭 / 토글 동작
- [ ] Hero(헤더) 카드 — 타이틀, 부제, 우측 위젯
- [ ] 메인 영역 분할 (좌측 표 + 우측 상세 / 또는 상하단 분할)
- [ ] 우측보기 / 하단보기 layout switch 토글 존재 여부
- [ ] 우측보기 시 액션 버튼 wrap 동작

## 2. 사이드바 / 네비게이션
- [ ] 브랜드 로고 텍스트
- [ ] 메뉴 항목 개수와 라벨 (mock vs 외부)
- [ ] 현재 페이지 활성 표시
- [ ] 사이드바 하단 (역할 스위치 / 사용자 정보)

## 3. 헤더 / Hero
- [ ] 타이틀 텍스트
- [ ] 부제(target명 / kicker / 컨텍스트 정보)
- [ ] 우측 사용자 메뉴(드롭다운: 사용자설정/로그아웃)
- [ ] document.title 형식

## 4. 툴바 (검색·필터·액션)
- [ ] 검색 input — placeholder, 카테고리 분리 여부
- [ ] 상세필터 버튼 + 활성 강조
- [ ] 전체 카운트 / 표시 범위 / 선택 카운트 캡션
- [ ] 액션 버튼 종류 (수정·삭제·조치계획작성·내보내기·이행점검 등)
- [ ] 권한별 버튼 가시성 (관리자 전용 등)

## 5. 상세 필터
- [ ] 필터 항목 (검출타입/담당자/상태 등)
- [ ] 검색 가능 여부
- [ ] 다중 선택 여부
- [ ] 적용/초기화 버튼

## 6. 테이블 컬럼 / 정렬
- [ ] 컬럼 개수와 순서
- [ ] 컬럼 헤더 텍스트
- [ ] 정렬 가능 컬럼 ↕ 표시
- [ ] 정렬 동작 시 URL 반영 여부

## 7. 페이지네이션 / 페이지 사이즈
- [ ] 페이지 사이즈 옵션 (mock 5/10/20 vs 외부 5/10/20/50/100 등)
- [ ] 페이징 컴포넌트 형식 (1, 2, ..., N)
- [ ] URL `?page=N&pageSize=N` 반영
- [ ] 페이지 변경 시 선택 상태 유지/초기화

## 8. 행 선택 / 일괄 액션
- [ ] 전체 선택 체크박스
- [ ] 페이지 전체 선택 시 "목록에서 총 N개 선택" 배너
- [ ] 일괄수정 모달 — 토글 컨트롤(상태/의견/담당자)
- [ ] 모달 내 상태 컨트롤 (그리드 vs dropdown)
- [ ] 의견 textarea 글자 수 카운터
- [ ] 담당자 변경 dropdown
- [ ] 적용 버튼 활성 조건

## 9. 삭제 모달
- [ ] 모달 제목 / 메시지 / 경고문구
- [ ] 취소 / 삭제 버튼 색상

## 10. 우측 상세 - 상태 카드
- [ ] 카드 제목
- [ ] 변경일시 표시 라벨 / 형식
- [ ] 변경 히스토리 시계 아이콘
- [ ] 닫기(×) 버튼
- [ ] 상태 옵션 개수와 라벨 (mock vs 외부 — `조치완료` 같은 누락 주의)
- [ ] 의견 textarea
- [ ] 담당자 picker
- [ ] 저장 버튼

## 11. 우측 상세 - 보조 카드 (PII / 검출내용 / 상세값 등)
- [ ] 카드 분리 구조 (mock 단일 vs 외부 다중)
- [ ] 검색 / 정렬 / 페이징
- [ ] 필드별 복사 아이콘
- [ ] 인라인 액션 버튼 (이행점검 등)

## 12. 변경 이력 모달
- [ ] 모달 제목
- [ ] 컬럼 (변경일시/변경자/상태/의견/담당자 — 누락 주의)
- [ ] 페이지네이션
- [ ] 일시 표기 형식
- [ ] 변경자 형식 (역할명 vs 사용자 표시명)

## 13. 역할 기반 권한
- [ ] 역할 스위치 위젯 존재
- [ ] 권한별 버튼/메뉴 분기

## 14. URL / 라우팅 파라미터
- [ ] mock 파라미터 (`dbId`, `target`, `detectType`, `detectionId`, `detectionQuery`, `piiQuery` 등) 지원 여부
- [ ] 외부 신규 파라미터 (`page`, `pageSize`, `sortKey`, `sortDir`, `panelId` 등)
- [ ] 새로고침/북마크 시 상태 복원

## 15. 시각 디자인
- [ ] 메인 컬러
- [ ] 카드 라운드/그림자
- [ ] 상태 칩 컬러 (미확인/조치필요/조치완료/제외 등)
- [ ] 칩(chip) UI

## 16. 실제 동작 검증 (영속 변경)
사용자 승인하에 직접 실행:
- [ ] 단건 패널 저장 — 토스트 / 변경이력 row 추가 / 마지막 변경 시간 갱신
- [ ] 일괄수정 적용 — 토스트 문구
- [ ] 이행점검 — 동기/비동기, 토스트
- [ ] 내보내기 — 토스트, 네트워크 호출 (`/api/v2/...`), 다운로드 트리거
- [ ] 조치계획 작성 — 라우팅 방식 (sessionStorage vs URL 쿼리)
- [ ] 삭제 — 토스트 색상, 행 즉시 제거, 카운트 갱신

## 17. 콘솔/네트워크
- [ ] 콘솔 에러/경고
- [ ] 주요 API 엔드포인트 (`/api/v2/...`) 형식
