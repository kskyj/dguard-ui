## 7. 우선순위별 권장 조치

### 🔴 High — 출시 전 반드시 보완 필요

1. **mock에 `등록 / 벌크 등록 / 점검대상 수정` 기능 설계 추가** — 외부에는 이미 슬라이드 패널 형태로 구현되어 있으나 mock은 목록+모달 수준에 머물러 있음. 가이드/사양 불일치 크므로 mock 갱신이 우선.
2. **상태 체계 정렬** — mock 4종(READY/RUNNING/COMPLETED/FAILED) → 외부 5종(대기/점검중/점검완료/**중지됨**/점검오류). "중지됨(STOPPED)" 추가 + 오류 라벨을 "점검오류"로 통일.
3. **`검색 수` 컬럼 처리 방침 확정** — 외부는 미제공. mock에 유지한다면 외부 구현 누락으로 분류해 업체에 반영 요청, 아니면 mock에서 제거.

### 🟡 Medium — UX/연계 개선

4. **상세필터의 `라벨 검색` 필드 누락 보완** — mock 설계대로 라벨 검색을 상세필터에 포함하거나, 외부처럼 검색 카테고리 방식으로 mock을 정렬.
5. **라벨 일괄수정 모달에 `현재 라벨 summary`를 외부에도 요청** — 수정 전 현재 상태를 시각화해야 실수 방지.
6. **URL 라우팅을 mock에도 도입 검토** — 정렬/페이징/필터/선택 카테고리를 URL에 반영하면 대시보드 공유, 책갈피, 뒤로가기 UX가 개선됨.
7. **검출건수 링크 대상 통일** — mock(DB별) vs 외부(스케줄별) 분기에 대한 공통 정의 필요.

### 🟢 Low — 시간 여유 시

8. **점검 시작 모달의 2줄 안내문**을 외부에도 반영 (또는 mock에서 단문으로 축약) — 사용자 이해도 균형.
9. **헤더 사용자 메뉴 dropdown** — 외부도 헤더에 프로필/설정/로그아웃 메뉴를 제공해 UX 일관성 확보.
10. **브레드크럼 추가** — 외부 헤더에 `점검대상 > DB 점검 대상 목록` 형태 보강.
11. **토스트 위치 정렬** — mock 우상단 vs 외부 우하단. 조직 표준 결정.

### 9.1 추가 발견 사항

- 외부는 라벨 일괄수정 모달에서 **Enter 키로 토큰화**가 필수. 입력만 하고 확인 누르면 반영 안 됨 — mock에도 동일 규칙.
- 외부의 라벨 autocomplete dropdown은 **이미 추가된 라벨도 중복 제안**함 → 중복 추가 시 서버가 dedupe 처리(실제로 추가 2회 해도 1개만 저장).
- 외부의 정렬은 일부 컬럼(`targetDisplayId`)에서 **3rd click 시 sort 해제** → `recentDetectionCount`는 asc↔desc 토글 — 컬럼별 다른 패턴 관찰 (검증 환경 데이터가 7건이라 미세 확인 한계).
- 외부의 `점검 시작` 버튼은 **선택 없어도 primary 스타일이지만 클릭해도 아무 일 안 일어남** (모달이 뜨지 않음). 1건 이상 선택해야 모달 노출됨.

### 9.3 검증 후 외부 시스템에 남긴 변경 흔적 (참고)

- **DB-082** (MariaDB-92) 라벨: 비교 테스트 중 `비교테스트` 태그 추가 → 제거 왕복 완료. 최종 상태는 원본 그대로 `중요` 1종만 유지.
- 그 외 등록/수정/삭제는 모두 **취소** 또는 **미실행**하여 외부 데이터에 잔여 변경 없음.
