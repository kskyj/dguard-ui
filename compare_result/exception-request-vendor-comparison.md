## 2. 전체 일치도 요약

| 정렬 | 일치 | 모든 컬럼 정렬 표시기(↑↓) 외부도 동일 |
| 페이지 사이즈 / 페이징 | 부분일치 | mock [10/20/50/100] → 외부 [5/10/20/50/100], 기본값 mock 10 → 외부 20 |
| 행 선택 / 일괄 액션 | 부분일치 | 체크박스·일괄 승인/거부 동일, mock의 `페이지/전체 목록 선택` 배너 없이 단순 "N건 선택" 표시 |
| 승인 모달 | 부분일치 | 기본 UX 동일(500자), 타이틀/caption/확인버튼 라벨 문구 차이 |
| 거부 모달 | 부분일치 | 승인 모달과 동일 구조, 거부 버튼 danger 색상 |

| 승인 후 자동 탭 전환 | 누락 | mock은 승인 후 processed 탭으로 자동 이동 → 외부는 신청목록 탭 유지 |
| 시각 디자인 | 부분일치 | 보라 accent·pill tab·카드 레이아웃 방향성 유사, 디테일 차이 존재 |

## 7. 우선순위별 권장 조치

### 🔴 High — 출시 전 반드시 보완 필요
1. **일반사용자 `제외신청취소` 기능 구현** — 본인이 실수로 신청한 건을 관리자 개입 없이 취소할 수 있어야 함. Mock은 user 역할 선택 시 툴바에 `제외신청취소` 버튼이 노출되고, 선택 후 클릭 시 대상 건이 `UNCHECKED`로 복귀하는 흐름을 제공.
2. **승인·거부 후 `processed` 탭 자동 전환** — 관리자가 작업 결과를 바로 확인할 수 있도록 mock의 `state.currentTab = "processed"` 흐름을 재현.

5. **페이지 필터 전체 선택 배너** — "페이지에서 N개 선택 → 검색결과 전체 M개 선택" 2단계 선택 UX.

11. **탭 이름 일관성** — "신청목록/처리결과"와 "제외신청목록/제외처리결과" 중 제품 용어 통일.

### 9.1 추가 발견 사항

- 외부 상세필터의 `제외 취소` 옵션은 mock 정책과 다름. mock은 `cancelRows`에서 `status = "UNCHECKED"`로 되돌려 `getVisibleRows`에서 제외되어 `processed` 탭에 나타나지 않음. 외부는 취소 이력을 별도 상태로 유지하는 것으로 보임 — 정책 합의 필요.
- `제외 승인`/`제외 거부`에서 외부 모달의 확인 버튼은 의견이 빈 문자열이면 disabled 처리됨(mock과 동일). 텍스트 입력 후 `19/500` 카운터가 올바르게 동작함.
- 외부 신청목록 탭 헤더 우측에 선택 시 `1건 선택 [승인][거부]` 핀이 나타나는 구성이며, mock처럼 `페이지 N개 / 전체 M개` 2단계 선택 배너는 없음.
- 처리결과 탭의 `처리상태` 값이 `제외`, `제외거부`, `제외 취소` 세 종류로 운영되는 듯함(mock은 `EXCLUDED`/`EXCLUSION_REJECTED` 2종만).

### 9.2 권장 추가 조치

- 🔴 `제외신청취소` UI/API를 외부 구현에 추가하거나, mock에서 기능을 제거하고 외부와 동일하게 "관리자 거부" 흐름만 유지하도록 정책 재확정.
- 🟡 `제외 취소` 상태를 mock에도 도입하여 외부 정책과 일치시키거나, 외부 구현을 `UNCHECKED` 복귀 정책으로 통일.
- 🟡 토스트 메시지/모달 문구 국문 가이드 정립 (건수 포함 여부, 액션 완료 어미 등).
