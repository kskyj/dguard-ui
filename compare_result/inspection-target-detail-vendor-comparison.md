## 7. 우선순위별 권장 조치

### 🔴 High — 출시 전 반드시 보완 필요
1. **실패 사유 노출 통로 복구** — 외부에 실패 행 (i) 버튼/모달이 없어 운영자가 실패 원인을 빠르게 볼 수 없음. mock의 모달 또는 동등한 toast/popover를 외부에 추가하거나 mock에서도 실패 사유를 다른 위치(상세 페이지 헤더 알림 등)로 빼는 결정 필요.
2. **점검대상 정보 카드의 Proxy/에이전트 상태** — connectivity 상태 pill을 카드 메인에 노출하고, 슬라이드 패널로만 가두지 말 것. 운영 1차 점검에서 가장 많이 보는 정보임.
3. **정책 계층 차이 정리** — mock은 SCHEMA_POLICY(스키마 단위 기본) 행을 지원하지만 외부 등록 모달은 "DB 전체 / 개별 테이블"만 라디오로 제공. 스키마 단위 기본 설정 행이 외부에서 어떻게 표현·생성되는지 확인 후 mock 또는 외부 중 한쪽을 맞출 것.
4. **카드 제목 / 검출 범위 라벨 용어 통일** — "테이블 정보 ↔ 검사 규칙", "업데이트된 ROW만 검출 ↔ 증분 검출". 사용자 매뉴얼/사이드바/툴팁까지 한 사전으로 정리.

### 🟡 Medium — UX/연계 개선
5. **점검 시작 흐름 일치화** — 외부의 마법사(`/search/detection/step2`)에 mock UI를 어떻게 연결할지 결정. mock의 "점검" 버튼은 현재 동작이 비어 있어 mock 기준으로는 고도화 필요.
6. **검출건수 클릭의 의미 합의** — mock(DB+카테고리) vs 외부(스케줄 단위). 둘 중 어느 진입을 표준으로 할지 결정. 둘 다 지원하려면 행 안에 두 개의 진입을 둘 것.
7. **검사 규칙 행 단위 삭제** — mock에 누락. 외부와 동등한 삭제 액션 추가하고 확인 모달 도입.
8. **인프라 담당자/그룹명 카드 노출** — 외부에는 카드에서 사라짐. 운영 책임자 식별이 필요하면 카드 또는 호버에 다시 추가.
9. **사이드바 메뉴 1:1 정합** — 외부의 정책관리(프로파일·예외필터)/대상관리(서버·계정·그룹·이미지·토큰·라이선스)/보안관리(사용자·그룹·접근권한·감사로그)와 mock 사이드바가 일치하는지 점검(검출목록 페이지 검증 시 확인된 라우트 외 추가 진입).

### 🟢 Low — 시간 여유 시
