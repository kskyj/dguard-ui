## 7. 우선순위별 권장 조치

### 🔴 High — 출시 전 반드시 보완 필요

1. **외부 "수정" 버튼 라우팅 버그** — `/inspection/schedule/new` → `/app/schedule/edit?id=SCH-XXXX` (또는 `?panelId=240`)로 수정. 현재 404 나와서 편집 진입 자체가 막혀 있음.
2. **상태 라벨 일관성** — 요약 칩 "실행중 / 완료 / 중단"과 행 내 상태 칩 "점검중 / 완료 / 중단"이 서로 다름. 한 쪽으로 통일 필요.
3. **역할 기반 권한** — mock에서 일반사용자는 중단/재시작/삭제 비활성인데 외부는 상시 활성. role 분기 도입 또는 최소한 삭제 버튼만이라도 관리자 제한.
4. **스케줄 등록 페이지 연결** — "스케줄 등록 페이지 (준비 중)" 플레이스홀더 해소. mock도 빈 핸들러이므로 양쪽 모두 신규 구현 필요.

### 🟡 Medium — UX/연계 개선

5. "중지" vs "중단" 용어 통일 — 프로젝트 네이밍 가이드 결정 후 mock·외부 모두 한 쪽으로.
6. 상세필터 다중 선택 지원 — 실제 사용자는 "점검중 + 대기" 같은 복합 필터를 원할 가능성이 높음.
7. breadcrumb 추가 검토 — 점검대상과의 연계성을 시각적으로 드러냄.
8. Hero 우측 사용자 메뉴 드롭다운 일관화 — 다른 페이지(예: 검출목록)에도 없다면 전사 결정이지만 있다면 일관성 문제.
9. 재시작 후 상태 전이 — mock은 "WAITING"으로만 전이하는데 외부는 즉시 "RUNNING". UX 관점에서 "즉시 실행"인지 "예약"인지 명확히 할 필요.

### 🟢 Low — 시간 여유 시

10. 점검자 badge 스타일 — 이메일 전체가 길면 레이아웃 깨질 수 있음. ellipsis 또는 이니셜 뱃지 고려.
11. 점검대상 레벨 다양성 — mock의 DB/스키마/테이블/컬럼 4종 모두 외부에서도 렌더되는지 다른 스케줄로 확인.
12. 캡션 포맷 — mock의 `전체 N건 / a-b 표시중 / N건 선택됨` 조합을 외부에도 반영할지.
